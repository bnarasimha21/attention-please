# Trending 02 — Context engineering

**Title:** Context engineering: the skill that's replacing prompt engineering
**Duration target:** 1:50
**Word count:** ~260 words (~140 wpm, tightened preview cut)
**Topic:** Context engineering — optimizing everything *around* the prompt
**Track:** Trending (pairs with main-track "What is an AI agent?" + T01 harness)

## NARRATION SCRIPT
*(Read this aloud to record your audio. Tone cues in [brackets].)*

---

### SCENE 1 — Hook [0:00–0:13]

[Calm, a little provocative — pull them in]

Watch this. Someone types the *perfect* prompt.

Crisp. Specific. Textbook.

And the AI gives back… garbage.

Here's the twist: the prompt was fine. The space *around* it was empty.

The magic was never the question.

---

### SCENE 2 — Prompt vs context [0:13–0:26]

[Explanatory, clear]

So here's the shift.

Prompt engineering tweaks one tiny box — the question you type.

Context engineering fills the whole stage *around* it: everything the model can see before it answers.

One is a sentence. The other is the entire room the model thinks inside.

---

### SCENE 3 — What goes into context [0:26–0:42]

[Building — list it out]

What's actually on that stage? Layer by layer.

System instructions — who the model is.
Retrieved documents — the facts it needs.
Memory — what happened before.
Tools — what it can do.
Examples — what good looks like.

And only *then*, the user's question.

The prompt is the last brick. Context is the whole building.

---

### SCENE 4 — The window is finite [0:42–0:56]

[Matter-of-fact, landing a key concept]

But here's the catch: the stage has a fixed size.

The context window only holds so much.

Dump in junk — old chatter, irrelevant files — and it crowds out the facts that actually matter.

Signal drowns in noise. Same window. Worse answer.

---

### SCENE 5 — Curation is the skill [0:56–1:12]

[Slightly serious — this is the craft]

So the real skill isn't *more* context. It's the *right* context.

Bad context: dump everything in, messy and bloated. The model guesses.

Good context: curated, ordered, compressed. The model nails it.

Same model. Same question. The only difference is what you put around it.

---

### SCENE 6 — Right info, right place, right time [1:12–1:26]

[Practical — the technique]

The pros don't preload everything.

They pull the *exact* doc the model needs, at the *moment* it needs it.

Just-in-time. The right information, in the right place, at the right time.

A clean window beats a full one.

---

### SCENE 7 — Why it matters [1:26–1:38]

[Practical — the big takeaway]

Here's why this changes everything.

The best model in the world, fed bad context, loses to a weaker model with great context.

The model is the engine. Context is the fuel — and the map.

---

### SCENE 8 — Recap + CTA [1:38–1:50]

[Warm, direct — close strong]

So, to recap.

Your prompt is maybe ten percent. Context is the system around it.

Curate it. Order it. Deliver it just in time.

Next time: context *rot* — why long chats slowly get dumber.

If this made something click, hit subscribe — it really helps.

See you in the next one.
